# 🏥 متجر الطب العالمي - دليل التشغيل الكامل

## 📋 نظرة عامة

متجر الطب العالمي هو منصة إلكترونية متكاملة لبيع الأجهزة الطبية في العراق، مع نظام دفع إلكتروني آمن وواجهة مستخدم عربية احترافية.

## ✨ المميزات الرئيسية

- 🛍️ **متجر إلكتروني كامل** مع إدارة المنتجات والطلبات
- 🔐 **نظام مصادقة آمن** باستخدام JWT
- 💳 **نظام دفع متكامل** مع Stripe والدفع البنكي
- 📊 **قاعدة بيانات MongoDB** قوية ومرنة
- 🌍 **واجهة عربية RTL** احترافية
- 📝 **نظام تسجيل أخطاء** متقدم
- 🧪 **اختبارات شاملة** لضمان الجودة
- 🚀 **نشر تلقائي** وسهل

## 🛠️ التقنيات المستخدمة

### الواجهة الخلفية (Backend)
- **Node.js** + **Express.js** - إطار العمل
- **MongoDB** + **Mongoose** - قاعدة البيانات
- **JWT** - المصادقة والتفويض
- **Stripe** - معالجة الدفعات
- **Winston** - تسجيل الأخطاء
- **Jest** - الاختبارات

### الواجهة الأمامية (Frontend)
- **HTML5** + **CSS3** + **JavaScript**
- **Bootstrap** - تصميم متجاوب
- **Font Awesome** - الأيقونات
- **Google Fonts** - الخطوط العربية

### الأمان والأداء
- **Helmet** - حماية الرأس
- **CORS** - مشاركة الموارد المتقاطعة
- **Rate Limiting** - الحد من الطلبات
- **Compression** - ضغط الاستجابات
- **XSS Protection** - الحماية من هجمات XSS

## 🚀 خطوات التثبيت والتشغيل

### 1. المتطلبات الأساسية
- Node.js (الإصدار 18 أو أحدث)
- npm (الإصدار 9 أو أحدث)
- MongoDB (محلي أو سحابي)

### 2. تثبيت المشروع
```bash
# استنساخ المشروع
git clone <repository-url>
cd medical-store

# تثبيت الاعتماديات
npm install
```

### 3. إعداد متغيرات البيئة
```bash
# نسخ ملف البيانات
cp .env.example .env

# تحرير الملف
nano .env
```

أضف المتغيرات التالية:
```env
NODE_ENV=development
PORT=5000
MONGODB_URI=mongodb://localhost:27017/medical-store
JWT_SECRET=your-super-secret-jwt-key
JWT_EXPIRE=7d
STRIPE_SECRET_KEY=sk_test_your-stripe-key
STRIPE_PUBLISHABLE_KEY=pk_test_your-stripe-key
FRONTEND_URL=http://localhost:3000
```

### 4. تشغيل المشروع
```bash
# للبيئة التطويرية
npm run dev

# للبيئة الإنتاجية
npm start
```

### 5. النشر التلقائي
```bash
# جعل السكربت قابلاً للتنفيذ
chmod +x deploy.sh

# تشغيل النشر
./deploy.sh
```

## 🔐 الأمان

### تدابير الأمان المطبقة
- **تشفير كلمات المرور** باستخدام bcrypt
- **توكنات JWT** منتهية الصلاحية
- **الحماية من XSS** و CSRF
- **التحقق من صحة المدخلات**
- **الحد من عدد الطلبات** (Rate Limiting)
- **أمان الرأس** باستخدام Helmet

### أفضل الممارسات
- تحديث الاعتماديات بانتظام
- استخدام متغيرات بيئة للبيانات الحساسة
- تفعيل HTTPS في الإنتاج
- نسخ احتياطي منتظم لقاعدة البيانات

## 💳 نظام الدفع

### طرق الدفع المدعومة
- **بطاقات الائتمان/الخصم** (Stripe)
- **التحويل البنكي** المباشر
- **المحافظ الإلكترونية** (قريباً)

### إعداد Stripe
1. إنشاء حساب في [Stripe](https://stripe.com)
2. الحصول على مفاتيح API
3. إضافتها إلى متغيرات البيئة
4. اختبار الدفع باستخدام وضع الاختبار
